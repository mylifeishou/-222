# GitHub 페이지 구조 계획

## 페이지 구조

### 1. 메인 랜딩 페이지 (index.html) - 단일 페이지 애플리케이션

#### 1.1 히어로 섹션 (Hero Section)
- **목적**: 첫인상, 핵심 가치 제안 전달
- **내용**:
  - 메인 헤드라인: "아파트 매수 타이밍, AI가 판단해드립니다"
  - 서브 헤드라인: Value Positioning 한 문장
  - CTA 버튼: "무료로 시작하기", "GitHub에서 보기"
  - 히어로 이미지/일러스트: 차트나 대시보드 시각화

#### 1.2 문제 정의 섹션 (Pain Points)
- **목적**: 사용자의 공감대 형성
- **내용**: PRD의 4가지 Pain Points 시각화
  - 정보 조각화
  - 예측·인사이트 부족
  - 거래 속보 인지 어려움
  - 비교 기능 부족

#### 1.3 해결책 섹션 (Solution / Features)
- **목적**: 핵심 기능 소개 및 차별점 강조
- **내용**: MVP 6가지 핵심 기능 상세 설명
  1. 단지 검색 및 실거래가 히스토리 차트
  2. AI 기반 가격 예측
  3. 단지 인사이트 자동 생성
  4. 변화 감지 알림
  5. 단지·평형 비교 기능
  6. 관심 단지 관리

#### 1.4 사용 사례 섹션 (Use Cases)
- **목적**: 타겟 고객별 맞춤 메시지 전달
- **내용**:
  - 예비 실거주자 케이스
  - 개인 투자자 케이스
  - 중개업자 케이스

#### 1.5 기술 스택 섹션
- **목적**: 기술적 신뢰도 구축
- **내용**: Next.js, Supabase, OpenAI 로고 및 설명

#### 1.6 시연/스크린샷 섹션
- **목적**: 실제 기능 시각화
- **내용**:
  - 대시보드 스크린샷
  - 차트 시각화 예시
  - AI 인사이트 예시
  - 비교 기능 UI

#### 1.7 로드맵 섹션 (선택사항)
- **목적**: 향후 계획 공유
- **내용**: Phase 1 (MVP), Phase 2, Phase 3 요약

#### 1.8 시작하기 섹션 (Final CTA)
- **목적**: 행동 유도
- **내용**:
  - "지금 시작하세요" 헤드라인
  - CTA 버튼: "무료로 시작하기", "GitHub에서 보기", "문서 보기"
  - 소셜 링크 (선택사항)

### 2. 네비게이션 구조
- **고정 헤더**: 로고, 메뉴 (Features, About, GitHub), CTA 버튼
- **앵커 링크**: 각 섹션으로 부드러운 스크롤
- **모바일**: 햄버거 메뉴

### 3. 푸터
- 프로젝트 설명 요약
- GitHub 링크
- 문서 링크
- 저작권 정보

```
landing-page/
├── index.html          # 메인 랜딩 페이지

├── js/

├── images/
│   └── screenshots/   # 스크린샷 이미지
├── plans/             # 계획 문서 (현재 위치)
└── README.md          # 랜딩 페이지 설명
```
