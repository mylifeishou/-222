# GitHub 페이지 계획 개요

## 프로젝트 개요
**실거래가 + 예측가격 + 인사이트 플랫폼**

아파트 실거래가·트렌드·예측가격·단지 인사이트를 하나의 화면에서 제공하고, 거래 변화 시점을 자동으로 알려주는 **타이밍 기반 의사결정 솔루션**입니다.

## 프로젝트 목표

### Primary Goal
사용자가 **아파트 매수·투자 타이밍을 명확하게 판단**할 수 있게 하는 것

### Secondary Goals
- 단지별 실거래 흐름을 직관적으로 시각화
- AI 기반 예측으로 불확실성을 줄임
- 의미 있는 변화 발생 시 자동 알림
- 사용자별 맞춤형 인사이트 제공

## 핵심 가치 제안 (Value Positioning)
**"우리 제품은 '언제 사고 팔아야 할지 판단하기 어려운' 사용자들이 기존의 단순 실거래 조회 앱보다 더 빠르고 정확하게, 실거래 추세·예측가격·변화 시점 인사이트를 한 화면에서 확인해 타이밍 의사결정을 명확하게 하도록 돕는다."**

## 문제 정의 (Pain Points)

예비 실거주자 / 투자자 / 중개업자가 겪는 문제:

1. **정보 조각화**: 여러 앱(KB부동산·호갱노노·부동산114)을 왔다 갔다 해야 해서 흐름을 파악하기 어려움
2. **예측·인사이트 부족**: 단순 데이터만 있고, **"지금 사고 팔 타이밍인가?"** 를 판단해주는 기능 부재
3. **거래 속보 인지 어려움**: 기존 알림은 단순 실거래 등록 알림 수준 → 의미 있는 변화(급등/급락/특정 평형 거래 등)를 잡아내지 못함
4. **비교 기능 부족**: 단지끼리, 평형끼리, 지역끼리 비교가 어려움

## 타겟 고객

### 1순위 – 예비 실거주자 (30~40대)
- 내 집 마련 준비 중
- 단지별 흐름과 타이밍에 가장 민감
- 기존 앱 많이 사용하나 '결정'이 어려움

### 2순위 – 개인 아파트 투자자
- 단지 비교 + 변동 트렌드 파악 욕구 강함
- 커뮤니티 활동 활발, 전문화된 분석 니즈 높음

### 3순위 – 중개업자
- 상담 시 최신 데이터·인사이트가 필요
- "한 화면 요약" 기능 강한 니즈

## 핵심 기능 (MVP)

### 🔥 MUST HAVE
1. 단지 선택 + 실거래가 히스토리 차트 (1/3/6/12개월)
2. OpenAI 기반 가격 예측(단지·평형별)
3. 단지 인사이트 자동 생성 (AI 요약)
4. 의미 있는 변화 감지 알림 (Rules + AI Hybrid)
5. 단지·평형 간 비교 페이지
6. Supabase 사용자 인증 + 관심 단지 저장

## 기술 스택
- **Frontend**: Next.js (Server Components + Client Components 혼합)
- **Backend**: Next.js API Routes
- **Database/Auth**: Supabase (데이터, Auth, Storage)
- **AI**: OpenAI (예측 + 인사이트 생성)

## GitHub 페이지 목적
프로젝트를 소개하고, 핵심 가치와 기능을 명확하게 전달하여 개발자와 잠재 사용자에게 프로젝트의 가치를 알리는 것
